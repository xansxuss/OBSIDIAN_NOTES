**指標自增（Pointer Increment）** 是記憶體操作中非常基礎且強大的觀念。
這背後涉及的是記憶體位址的計算。

以下是針對指標自增的關鍵點整理：

### 1. 核心邏輯：位址偏移量

指標自增（如 `p++`）並非單純將位址加 1，而是根據**指標指向的資料型別（Data Type）的大小**來移動位址。

$$\text{New Address} = \text{Old Address} + (1 \times \text{sizeof(type)})$$

- **char 指標**：`p++` 位址增加 1 byte。
    
- **int 指標**：`p++` 位址增加 4 bytes（取決於系統架構）。
    
- **自定義 Struct 指標**：`p++` 位址會跳過整個結構的大小。
    

---

### 2. 常見寫法與優先權

在處理陣列或緩衝區時，指標自增常與解引用（Dereference）結合。這裡的優先權（Precedence）非常關鍵：
|**運算式**|**說明**|
|---|---|
|`*p++`|**最常用**。先取 `*p` 的值，然後將指標 `p` 移向下一個位址。|
|`(*p)++`|指標不變，將 `p` 指向的**內容值**加 1。|
|`*++p`|指標先移向下一格，再取該位址的內容。|
|`++*p`|同 `(*p)++`，但先加值再回傳結果。|

### 3. 工程實踐中的注意事項

- **陣列越界（Out of Bounds）**：自增過頭會指向未定義的記憶體區域，這是 Segment Fault 的常見主因。
    
- **指標型別轉換（Type Casting）**： 如果你將一個 `int*` 強轉為 `char*` 再進行自增，你可以實現對記憶體底層 Byte-level 的細粒度操作（這在處理二進制協定或模型權重序列化時很常見）。
    
- **const 指標**：
    
    - `const int *p`：可以自增（`p++`），但不能修改內容（`*p = 10` 錯誤）。
        
    - `int * const p`：**不可以自增**，指標本身是唯讀的。
        

---

### 4. 範例程式碼 (C++)

``` cpp
#include <iostream>

int main() {
    int arr[] = {10, 20, 30};
    int *ptr = arr; // 指向 arr[0]

    std::cout << "初始值: " << *ptr << " 位址: " << ptr << std::endl;

    ptr++; // 指標自增，移動 sizeof(int) 個位元組

    std::cout << "自增後: " << *ptr << " 位址: " << ptr << std::endl;

    return 0;
}
```
