是一種用來解決**指派問題(Assignment Problem)**的經典演算法。這種問題通常可以描述為：有 nn 個工作和 nn 個工人，每個工人做每個工作的成本不同，目標是找到一個分配，使總成本最小（或總收益最大）。
## 1️⃣ 問題定義

給定一個 $n×n$ 的成本矩陣 $C = [c_{ij}]$，其中 $c_{ij}$ 是工人 $i$ 執行工作 $j$ 的成本。目標：

$\min \sum_{i=1}^{n} c_{i, \pi(i)}​$

其中 $π$ 是一個排列（每個工人只做一個工作，每個工作只分配給一個工人）。

---

## 2️⃣ 演算法直覺

匈牙利演算法主要思路：

1. **行減法**：每一行減去該行最小值，保證每行至少有一個零。
2. **列減法**：每一列減去該列最小值，保證每列至少有一個零。
3. **建立零的匹配**：
    - 嘗試選擇一些零，使得每行每列最多選一個零。
4. **覆蓋零**：
    - 計算最少行+列的覆蓋，使所有零被覆蓋。
    - 如果覆蓋行列數 = n，匹配完成。
    - 否則，調整未覆蓋的元素：
        - 找出未覆蓋最小值 m
        - 未覆蓋元素減 m，同時覆蓋交叉的元素加 m
        - 重複覆蓋過程
5. **找到最終匹配**：
    - 最終能形成一個最大匹配，對應最小成本。

---

## 3️⃣ 時間複雜度

- 標準匈牙利演算法的時間複雜度是 $O(n^3)$。
- 對於稀疏矩陣或最大匹配的變種，可以做一些優化。

---

## 4️⃣ Python 範例（最小化成本）

```python
import numpy as np 
from scipy.optimize 
import linear_sum_assignment  
# 假設成本矩陣 
cost_matrix = np.array([[4, 2, 8], [2, 3, 7], [3, 1, 6]])
row_ind, col_ind = linear_sum_assignment(cost_matrix)
print("匹配結果:", list(zip(row_ind, col_ind)))
print("最小成本:", cost_matrix[row_ind, col_ind].sum())
```

這裡 `linear_sum_assignment` 就是匈牙利演算法的實作。
---

## 5️⃣ 小結

- 匈牙利演算法專門用來解決**二分圖匹配中的指派問題**。
- 核心是「減行減列」+「覆蓋零」+「調整矩陣」。
- 可以用於最小化成本，也可以透過取負數改成最大化收益。
