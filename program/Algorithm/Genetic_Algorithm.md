**遺傳演算法 (Genetic Algorithm, GA)** 是一種模擬達爾文「物競天擇、適者生存」生物進化理論的隨機尋優演算法。

它特別適合處理傳統梯度下降法容易卡在局部最佳解（Local Optimum）的複雜、非線性問題。

---

## 🧬 GA 的核心流程

GA 的運作就像是一個生態系的演化過程，主要包含以下幾個關鍵步驟：

1. **初始化 (Initialization)**：隨機產生一群「染色體」（Chromosomes），每一條染色體代表問題的一個潛在解。
    
2. **適應度評估 (Fitness Evaluation)**：計算每個個體的適應度分數（Fitness Score）。分數越高，代表這個解越優秀。
    
3. **選擇 (Selection)**：根據適應度從族群中挑選父母輩。最常見的方法是**輪盤法 (Roulette Wheel Selection)**，分數越高被選中的機率越大。
    
4. **交配 (Crossover)**：將選出的父母染色體進行部分基因交換，期望產生具備父母優點的後代。
    
5. **突變 (Mutation)**：以極低的機率隨機改變後代的某些基因位元，這是為了維持**族群多樣性**，防止過早收斂。
    
6. **迭代 (Iteration)**：重複上述步驟，直到達到終止條件（如找到夠好的解或達到最大代數）。
    

---

## 🛠️ 關鍵算子詳解

對於 AI 工程師來說，調整這些參數（Hyperparameters）是優化 GA 效能的關鍵：

| **算子 (Operator)**  | **說明**                        | **目的**          |
| ------------------ | ----------------------------- | --------------- |
| **編碼 (Encoding)**  | 將解轉為二進位 (0/1) 或實數向量。          | 定義問題的搜索空間。      |
| **交配率 (Pc)**       | 控制多少比例的個體進行基因交換（通常為 0.6~0.9）。 | 促進優秀特徵的組合。      |
| **突變率 (Pm)**       | 隨機翻轉基因的機率（通常極低，如 0.001~0.01）。 | 跳出局部最佳解，探索新區域。  |
| **菁英保留 (Elitism)** | 直接將當前代最強的個體複製到下一代。            | 確保演化過程中不會丟失最優解。 |

## 🚀 GA 在 AI 與工程中的應用

在你的工作領域，GA 常被用於以下場景：

- **神經網路架構搜索 (NAS)**：自動尋找最優的 CNN/Transformer 層數與參數配置。
    
- **超參數調優**：當搜尋空間太大，無法使用 Grid Search 時，GA 是很棒的替代方案。
    
- **組合優化問題**：經典的旅行推銷員問題 (TSP)、排班問題或物流路徑規劃。


**混合遺傳演算法 (Hybrid GA)**，也就是俗稱的 **模因演算法 (Memetic Algorithm, MA)** 結合「全局搜索」與「區域優化」的優點。

簡單來說，GA 擅長在大範圍內「亂槍打鳥」尋找潛力區域，但它在細微調整（Fine-tuning）方面效率極差；而 MA 則是給了這些個體「學習」的能力。

---

## 🧠 核心概念：GA + Local Search

在生物學中，基因（Gene）是遺傳的，而模因（Meme）是透過學習與文化傳遞的。MA 的核心公式如下：

> **MA = GA (Evolution) + Local Search (Learning)**

在每一代的演化過程中，MA 會對個體進行「局部改進」。這通常遵循 **拉馬克學說 (Lamarckian Learning)**：個體在後天學習到的優良特徵，會直接寫回基因並遺傳給下一代。

---

## 🛠️ MA 的執行流程

與標準 GA 相比，MA 多了一個 **"Individual Improvement"** 的步驟：

1. **初始化**：產生初始族群。
    
2. **局部搜索 (Local Search)**：**[關鍵步驟]** 對每個（或部分）個體進行局部優化（例如使用爬山法 Hill Climbing、模擬退火 Simulated Annealing 或禁忌搜索 Tabu Search）。
    
3. **適應度評估**：計算經過局部優化後的得分。
    
4. **選擇/交配/突變**：執行標準 GA 算子。
    
5. **重複**：直到收斂。
    

---

## 📈 為什麼 MA 比純 GA 更強？

作為工程師，我們在處理現實世界的複雜優化（如 **車隊路徑規劃 VRP** 或 **超大規模積體電路佈局 VLSI**）時，MA 有以下優勢：

| **特性**   | **純遺傳演算法 (GA)** | **混合遺傳演算法 (MA)**       |
| -------- | --------------- | ---------------------- |
| **搜索策略** | 隨機全局搜索          | 全局搜索 + 精度微調            |
| **收斂速度** | 較慢，容易在最佳解附近震盪   | 較快，能迅速鎖定局部極值           |
| **解的品質** | 普通，難以達到極高精度     | 優異，通常能找到更精確的解          |
| **計算開銷** | 較低 (單次迭代)       | 較高 (因為多了 Local Search) |

## 💻 實作建議 (AI Engineer's Perspective)

如果你打算在專案中導入 MA，有幾個常見的陷阱與技巧：

### 1. 局部搜索的頻率 (Intensity vs. Efficiency)

不要對「所有」個體都進行深度的局部搜索，這會讓運算時間爆炸。常見策略是：

- 只對每一代中**前 10% 的菁英**進行 Local Search。
    
- 每隔 $N$ 代才執行一次大規模的局部優化。
    

### 2. 局部搜索演算法的選擇

- **爬山法 (Hill Climbing)**：最簡單，適合連續空間。
    
- **2-opt / 3-opt**：專門用於路徑優化（如 TSP），透過交換節點順序來消除交叉路徑。
    
- **梯度下降 (Gradient Descent)**：如果你的適應度函數是可微的，可以直接套用。
    

### 3. 多樣性維護

由於 Local Search 會把個體快速推向附近的極值，族群很容易失去多樣性。建議搭配較高的**突變率**或使用 **Restarts 機制**。
